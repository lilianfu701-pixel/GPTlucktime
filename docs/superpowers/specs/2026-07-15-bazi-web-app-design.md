# 八字命盘推演网站设计

## 1. 目标与边界

本设计定义第一版 Web 网站的交互和展示层。它建立在“静态八字命盘核心”之上：用户输入出生资料，系统完成时间校正和四柱推演，输出可核验的出生固定指标。

第一版的目标是让用户能清楚地录入资料、生成命盘、查看完整静态指标，并追溯每一项时间与历法依据。它不输出吉凶结论，不推断身强身弱，也不把动态流年结果混入出生固定命盘。

不属于第一版：

- 登录、账户、命盘保存和分享链接。
- 大运、流年、流月、流日及其解读。
- 居住地对动态推演的影响。
- 年家、月家、日家九星盘及方位判断。
- 自动化命理解读、建议或预测。

## 2. 用户路径

### 2.1 首页：先录入，再看命盘

首页是一个单页资料录入工作台，不预先展示计算结果。

输入区按信息来源分为三组：

1. 出生时间：公历日期、当地钟表时间、时间精度、夏令时歧义选择（仅在需要时显示）。
2. 出生地点：地点搜索、国家/地区、经纬度、IANA 时区。地点搜索确认后，坐标和时区对用户可见、可修改。
3. 生活地点：作为后续动态模块的预留资料，第一版仅采集并明确提示“不会改变出生固定命盘”。

提交按钮为“校验时间并生成命盘”。提交前客户端只做完整性和格式校验；最终时区历史、夏令时、节气和真太阳时校验由核心推演层完成。

### 2.2 结果页：独立命盘工作台

生成成功后跳转到带有输入摘要的独立结果页。用户可选择“修改出生资料”，返回首页重新计算；第一版不保存用户数据，刷新或离开页面后不保留资料。

结果页由四个区域组成：

1. 顶部摘要：出生资料、地点、时区、计算版本、主要警告状态。
2. 中央命盘：年、月、日、时四柱；天干、地支、藏干和对应十神。
3. 右侧指标面板：出生固定的命理指标，按可折叠分组展示。
4. 底部核验面板：时间校正、真太阳时、节气、儒略日和计算追溯信息。

移动端按“摘要、四柱、指标、核验”顺序纵向排布；桌面端采用中央命盘加右侧指标的双栏结构。重要结果不依赖悬浮提示或仅颜色表达。

## 3. 静态指标与九星边界

右侧指标面板必须包含以下出生固定信息：

- 日主与月令。
- 天干十神、地支藏干及各藏干十神。
- 五行分布：天干、藏干与合计计数；仅陈述事实，不评定旺衰。
- 纳音、空亡、十二长生。
- 干支关系：合、冲、刑、害、破、会局等，并注明参与的柱位。
- 神煞：依据固定且可版本化的规则表计算，展示规则来源标识。
- 命宫、身宫、胎元、胎息等扩展出生指标；每项须有明确算法版本后才启用。
- 九星气学：本命星与月命星，作为出生日期推导出的固定指标。

九星气学的“年家九星、月家九星、日家九星”和九宫盘随指定日期变化，因此归为 P1 动态模块。它们不会在第一版结果中伪装成出生固定结论，但会预留“动态九星”标签页入口并标示“即将推出”。

## 4. 信息架构与交互细节

### 4.1 指标分组

右侧面板按以下顺序显示：

1. 十神与藏干。
2. 五行与日主。
3. 命局基础：纳音、空亡、十二长生、命宫、身宫、胎元、胎息。
4. 干支关系。
5. 神煞。
6. 九星气学：本命星、月命星。

每个分组默认展示关键条目，可展开查看计算参与对象和简短的中性定义。规则、数据源和算法版本集中放在底部核验面板，避免把解释性文案与事实性结果混为一谈。

### 4.2 警告和不确定性

系统必须显式展示而不静默猜测以下情况：

- 出生时间不详、仅知道时辰或跨时辰边界。
- 夏令时的重复/缺失当地时间。
- 地点坐标或历史时区无法可靠确定。
- 距离节气交接点过近，且输入时间精度不足。
- 第三方地名解析与用户手动坐标不一致。

带有警告的结果仍可展示，但对应字段标注“需确认”，并在核验面板说明原因与建议补充的资料。

## 5. 技术架构

网站采用 Next.js App Router 与 TypeScript。命盘计算不写入页面组件，而通过一个薄的服务适配层调用纯函数核心。

建议目录职责：

- `app/`：首页、结果页和 API/服务入口。
- `components/intake/`：出生资料、地点和时间校正表单。
- `components/chart/`：四柱、指标分组、核验面板和警告组件。
- `lib/chart/`：界面数据映射、展示格式化与输入校验，不承载历法算法。
- `core/`、`engines/`：静态八字核心设计中定义的纯计算模块。

数据流：

1. 表单收集 `BirthInput` 与可选生活地资料。
2. 服务适配层校验并调用 `buildChartContext`。
3. 核心返回含 trace、versions、warnings 的结构化命盘上下文。
4. 界面映射层将计算结果转换为“摘要、四柱、指标组、核验项”视图模型。
5. 页面只渲染视图模型，不自行重新计算干支、节气或九星。

首版不接数据库。出生资料仅在本次页面会话中使用；不得在未告知用户的情况下记录或上传精确出生地点和时间。

## 6. 视觉方向

页面使用纸张、朱砂与玉石绿的浅色调，强调“可校验的历法工具”而非神秘化预测。中文标题使用有书卷气的衬线字体，正文以易读的无衬线字体呈现。四柱采用纵向字格和清晰的柱位标记；五行和警告同时使用文字、图形与颜色。

首页强调逐步录入与校验状态；结果页强调四柱中心视觉和右侧可扫描指标。页面动画仅用于进入结果、展开指标和显示校验完成，不影响阅读或数据确认。

## 7. 验收标准

- 用户可输入当地出生日期、时间、出生地，明确看到时区、经纬度和夏令时处理结果。
- 生成页正确呈现核心返回的四柱及全部已实现的静态指标。
- 右侧包含十神和九星气学的本命星、月命星，不把动态九星盘混入静态结果。
- 用户可查看真太阳时、经度校正、UTC、节气、JDN、规则版本和警告。
- 生活地资料不会改变第一版静态命盘，并在界面上明确说明。
- 桌面与移动端均可完整访问所有信息。
- 对异常时间、地点和节气边界给出明确警告而非无提示生成。
- 页面、适配层与核心计算均有对应自动化测试；核心算法保持静态核心设计规定的覆盖率目标。
